Featured Live on AWS

Fullstack Docker Application on AWS

A production-style multi-service web application running Express.js, Flask, and MongoDB as separate Docker containers — orchestrated with Docker Compose, deployed on AWS EC2, and automated with Jenkins CI/CD.

Problem Solved

Manual deployments were slow and error-prone. I automated the entire pipeline from GitHub push to live deployment on EC2 using Jenkins and webhooks — zero manual SSH.

Architecture

  • GitHub repo → Jenkins webhook trigger
  • Jenkins builds & pushes Docker images
  • Docker Compose orchestrates Express + Flask + MongoDB
  • AWS EC2 (Ubuntu) hosts the stack with Elastic IP
  • EBS volume for persistent MongoDB data

Challenges & Solutions

  • GPG Key Error: Jenkins install failed — fixed by updating the apt keyring manually
  • Docker Socket Permission: Jenkins user lacked Docker access — added to docker group
  • Node Offline Error: Disk threshold triggered node offline — adjusted Jenkins disk threshold config
  • Webhook breakage: EC2 IP changed on restart — solved with Elastic IP allocation
